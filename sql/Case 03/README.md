\# Case 03 - SQL Analytics Engineer



\## Objetivo



Resolução de um case prático de SQL focado em métricas de negócio, análise de funil, coortes, conversão, qualidade de dados e uso de funções analíticas.



\## Conceitos praticados



\* Agregações (`SUM`, `AVG`, `COUNT`)

\* Filtros condicionais (`FILTER`)

\* `JOINs`

\* `CTEs`

\* Window Functions (`LAG`, `DENSE\_RANK`, `SUM OVER`)

\* Métricas de negócio

\* Análise de conversão

\* Coortes

\* Data Quality / Data Validation



\## Estrutura do exercício



\### Nível 1 - Aquecimento



\* Ticket médio por segmento

\* Evolução mensal de leads

\* Tempo médio de ciclo de vendas



\### Nível 2 - Regras de negócio



\* Leads sem atividades registradas

\* Performance por vendedor

\* Contratos ativos há mais de 6 meses



\### Nível 3 - Análises analíticas



\* Ranking mensal de vendedores por MRR

\* Conversão por canal com comparação MoM

\* Comparação de performance vs média do time

\* Coorte de aquisição



\### Nível 4 - Desafios



\* Construção de funil completo

\* Detecção de anomalias de dados



\## Observações



\* As tabelas do case possuem chaves primárias únicas.

\* Os `DISTINCT` foram mantidos apenas em cenários onde os `JOINs` poderiam gerar multiplicação de registros.

\* Algumas soluções foram mantidas mesmo havendo formas mais enxutas de implementação, pois o objetivo do repositório é estudo e prática de diferentes abordagens SQL.

\* O foco principal foi clareza de lógica, legibilidade e consolidação dos conceitos utilizados em processos seletivos para Analytics Engineer.
